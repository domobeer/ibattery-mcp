Give your AI assistant a readout on every battery you own.

让你的 AI 助手看到你所有设备的电量。

ibattery-mcp is an MCP server that reads battery and charging status from your Mac, Bluetooth accessories, iPhone/iPad, and Apple Watch — so Claude Code, Claude Desktop, or Work Buddy can just tell you, every morning, what needs plugging in.

ibattery-mcp 是一个 MCP 服务器，能读取 Mac 本机、蓝牙外设、iPhone/iPad、Apple Watch 的电量与充电状态——这样 Claude Code、Claude Desktop 或 Work Buddy 每天早上就能直接告诉你，哪个设备该充电了。

Why two processes?

为什么是两个进程？

macOS won't let a bare MCP server touch Bluetooth.

macOS 不允许裸的 MCP 进程碰蓝牙。

macOS attributes Bluetooth's privacy check to whichever process is "responsible" for launching yours — never the MCP subprocess itself, since it's spawned directly by its host, not opened normally. So a small companion app owns Bluetooth instead, and the stateless MCP server just asks it questions over a local socket.

macOS 会把蓝牙隐私检查的责任归到启动你的那个"负责进程"身上——而 MCP 子进程从来都不是这个"负责进程"，因为它是被宿主直接拉起来的，不是正常打开的。 所以蓝牙访问权限交给一个独立的小型伴生 App 持有，无状态的 MCP server 只通过本地 socket 向它提问。
